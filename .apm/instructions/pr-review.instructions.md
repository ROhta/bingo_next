# Pull Request でのレビューについて

## 確認観点の優先順位

PR をレビューする際は、**PR 本文の `## 見てほしいところ` セクションに列挙された項目を最優先で確認する**。

- レビューを開始したら、まず PR 本文 (`gh pr view <pr> --json body -q .body` 等) を取得し、`## 見てほしいところ` の各項目を抽出する
- 抽出した項目は、レビュー観点として最初にチェックリスト化し、コードを読む前に着眼点として頭に入れる
- 当該項目に対する所見 (妥当 / 懸念あり / 追加質問) を、他のレビュー指摘より先にコメントとして残す
- `## 見てほしいところ` が空、または項目が `- [ ]` のテンプレ初期値のままの場合は、その旨を作者に確認してから一般観点でのレビューに進む

理由: `## 見てほしいところ` は PR 作者が特に意見・確認を求めている箇所であり、ここを後回しにすると「作者の関心事に答えていない」レビューになりやすい。先にここを潰してから自由観点での指摘に移ることで、レビューの価値が安定する。
